# 基本設計書

## 1. 画面一覧

- **トップ画面**
  - アプリケーションのタイトル
  - ゲームモード選択（ステップ1 / ステップ2）
  - ゲーム開始ボタン

- **ゲーム画面**
  - 現在の問題数 / 全問題数 (例: 3/10)
  - 現在のスコア
  - 問題再生ボタン
  - 回答用UI
    - ステップ1の場合: 「ドである」「ドではない」ボタン
    - ステップ2の場合: ピアノ鍵盤UI
  - フィードバック表示エリア

- **リザルト画面**
  - 最終スコア表示
  - トップへ戻るボタン

## 2. UI/UXデザイン

### 2.1. ピアノ鍵盤
- **範囲:** 1オクターブ分を表示する。
- **インタラクション:** クリック/タップで対応する音が鳴る。

### 2.2. フィードバック
- **正解時:** 「正解！」などのポジティブなメッセージを表示。
- **不正解時:** 正解の音名をテキストで表示し、同時に正解の音を再生する。

## 3. ゲームフロー

1.  ユーザーが**トップ画面**でプレイしたいゲームモード（ステップ1 or 2）を選択し、「ゲーム開始」ボタンを押す。
    - (初回操作時) `Tone.js`のオーディオコンテキストが初期化され、音源のローディングが開始される。
2.  **ゲーム画面**に遷移し、1問目が開始される。
3.  ユーザーが「問題再生ボタン」を押し、問題の音が再生される。
4.  ユーザーが対応するUI（ボタン or 鍵盤）を操作して回答する。
5.  回答後、フィードバックが表示され、スコアが更新される。
6.  次の問題へ自動的に進む。
7.  全10問が終了すると、**リザルト画面**に遷移する。
8.  リザルト画面で最終スコアを確認し、「トップへ戻る」ボタンでトップ画面に戻る。

## 4. 機能一覧

- **音源再生機能:**
  - 単音の再生
- **ゲームロジック機能:**
  - 2種類のゲームモード（ステップ1, ステップ2）
  - 問題の出題と正誤判定
  - 10問1セットのゲーム進行管理
  - スコア計算（正解: 10点, 不正解: 0点）
- **UI機能:**
  - 画面遷移（トップ ⇔ ゲーム ⇔ リザルト）
  - 1オクターブのピアノ鍵盤
  - ボタンUI
  - 結果表示
