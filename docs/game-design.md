# ゲームデザイン仕様書

## 1. MVP（実用最小限の製品）仕様

初期リリース版として、以下の仕様を実装する。

### 1.1. 実装レベル
- **ステップ1：基準音の定着**
- **ステップ2：白鍵の1オクターブ**

ユーザーが上記2つのレベルを選択してプレイできる。

### 1.2. ゲームルール
- **問題数:** 1ゲームあたり10問
- **スコア:** 1問正解につき10点。不正解の場合は0点（ノーカウント）。
- **ゲーム進行:** 10問すべてに回答するとゲーム終了。ゲームオーバーはない。
- **フィードバック:** 不正解の場合、正解の音名をテキストで表示し、正解の音を鳴らす。

---

## 2. ゲームの目的

絶対音感（またはそれに近い音高記憶能力）を鍛えるためのトレーニングを提供する。
単調なトレーニングにならないよう、ゲーム性を取り入れ、継続的な学習をサポートする。

## 3. レベルデザイン（トレーニングステップ）

ユーザーが段階的にスキルアップできるよう、以下のステップで難易度を設計する。

### ステップ1：基準音の定着
- **内容:** 「ド（C4）」の音だけを覚え、「ド」か「ド以外」かを答える。
- **UI:** ○×クイズ形式

### ステップ2：白鍵の1オクターブ
- **内容:** 中央の1オクターブ（ド〜ド）の白鍵のみを出題。
- **UI:** 画面上のピアノ鍵盤をクリックして回答する。

### ステップ3：黒鍵を含めた1オクターブ
- **内容:** 1オクターブ内の12音（白鍵・黒鍵）すべてを出題。
- **難易度調整:** 「3択」など選択肢を絞る機能も検討。

### ステップ4：音域の拡大
- **内容:** 2〜3オクターブに範囲を広げ、オクターブ違いの同じ音名（C, D, E...）を当てさせる。

### ステップ5：和音の分解
- **内容:** 2和音、3和音の構成音をすべて当てる。
- **UI:** 鳴った和音の構成音を、画面上の鍵盤で複数クリックさせる。

## 4. 追加機能アイデア

- **共感覚モード:** 音と色を関連付けて鍵盤に表示する機能。
- **タイムアタックモード:** 回答までの制限時間を設ける。
- **楽器の持ち替え:** ピアノ以外の楽器音（シンセ、ギターなど）に変更できる機能。
- **基準音確認ボタン:** いつでも基準となる音（例：「ド」）を鳴らせるボタンを配置する。
